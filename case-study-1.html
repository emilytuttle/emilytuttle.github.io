Overview

Role

UX designer and researcher leading the app design from conception to delivery. Responsibilities included user research, wireframing, prototyping, mockup creation, and presentation creation.

Project Goal

Tony's Restaurant was in need of a mobile app to provide regular customers with a more convenient mobile ordering experience. While websites are accessibile through phones, they don't probide the option to save ordering information. With a dedicated mobile app, users are able to save their order information to place repeat orders, but more importantly, save previous order billing information.

Target Audience

Ideal users include busy people who want a specific app for their favorite restaurant instead of using a website everytime. These users include people of various ages and backgrounds, and therefore must be accessibile to all.

Key challenges or constraints

Tony's Restaurant is a fictional restaurant, so previous designs from existing branding or websites could not be built upon. Design sheets were made from scratch, as were customer profiles. However, this allowed for total creative freedom. There was no existing budget for this project, so all user testing particpants were volunteers.

Research

Research Study Details

Research included competitive audits and persona creation to discover common user pain points.

Common Pain Points

  • - Not finding menus
  • - Not having information saved
  • - Difficult to edit orders
  • - Long Processes

Ideation

Sketching & Wireframing

Wireframes were created with a very simple user flow in mind, with the menu option front and center. User pain points included difficult to find menus, long processes, or difficult order editing, so the goal was to make these as simple as possible. Items could be added to the order from the menu page, to avoid looking for items twice.

User Testing Results

Round 1 User Testing Findings

-A more detailed confirmation screen would decrease confusion

-Multiple types of edit buttons were confusing, so only one should be shown

-Remove buttons were necessary to make the order confirmation screen more usable.

Round 2 User Testing Findings

-Animations were not high quality yet

-User flow was missing a profile edit option

-Users didn't like the size of the edit buttons

Conclusion

Tony's Restaurant now has a fictional mobile app to go with their fictional restaurant. In the future, the app can be engineered for mobile use, and have a web application built to go alongside it. The web application would not provide an option to save payment information, but it would remove the need to download an app for irregular or first-time customers.
